小结：模板方法的两种实现方式：
1.	使用继承的方式，抽象方法和具体实现的关系式在编译期间静态决定的，是类级的关系；
	使用Java回调，这个关系是在运行期间动态决定的，是对象级的关系。
2.	相对而言，使用回调机制会更灵活，因为Java是单继承的，而回调是基于接口的。
3.	使用继承会更简单，因为父类提供了实现（钩子方法），子类不想扩展，就不用覆盖；
	使用回调机制，回调的接口需要把所有可能被扩展的方法都定义进去，这就导致实现的时候，不管你要不要扩展，都要实现这个方法，哪怕什么都不做，只要调转模板中已有的实现，都要写出来。
	
	使用模板方法模式可以将代码的公共行为提取出来，达到复用的目的。而且，在模板方法模式中，是由父类的模板方法来控制子类中的具体实现。这样你在实现子类的时候，根本不需要对业务流程有太多的了解。
